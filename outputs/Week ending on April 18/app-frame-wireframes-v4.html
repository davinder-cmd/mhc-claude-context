  • Same modules: All 8 modules present, just more compact
  • Less scrolling: ~2 screens vs ~4-5 screens in v2/v3

Expand vs Navigate:
Module Behavior Rationale
Stats Expand inline (⌄) Quick glance data, no deep drill needed
Actions Expand inline (⌄) Show task list, "See all" links to full page
AI Insights Navigate (›) Full insight requires context + CTA
DCP Navigate (›) Deep content — lessons, progress, history
Habits Expand inline (⌄) Quick toggle checklist
Content Navigate (›) Opens content hub or specific article
Challenge Navigate (›) Leaderboard, team, full progress

3. Interaction Patterns

Chevron Affordances

Icon Meaning Behavior
⌄ Expand Content expands inline below card
⌃ Collapse Content collapses back to summary
› Navigate Opens full page / detail view
Expand vs Navigate decision: If the user can complete the task from the inline view (check a habit, see a stat), expand. If they need deeper context (read an article, view leaderboard), navigate.

Scroll Depth Comparison

Version Scroll Depth Cards Visible
v2/v3 (Full) ~4-5 screens All expanded
v4 (Condensed) ~1.5-2 screens All collapsed
v4 (Expanded) ~3 screens Mix of expanded/collapsed
Why condensed is better for mobile:
  • Users scan headlines, not details
  • Reduces scroll fatigue
  • Key info visible in summary
  • User controls information density

4. Version Comparison

Feature v2 (Full Parity) v4 (Condensed)
Mobile content All modules fully expanded All modules as scannable rows
Scroll depth ~4-5 screens ~1.5-2 screens (collapsed)
Information access Immediate — all visible Progressive — tap to reveal
Interaction model Scroll and scan Tap to expand/navigate
Key data visible All details Headlines + badges
User control None (fixed layout) High (expand what matters)
Pattern similar to Social feeds, news apps Personify, Apple Health, banking apps
When to use v4 (condensed):
  • Users check app quickly (glance at stats, check tasks)
  • Home is a dashboard, not a feed
  • Prefer density over depth on first view
  • Match Personify's scannable card pattern
When to use v2/v3 (full parity):
  • Content discovery is a primary goal
  • Users spend time browsing (not just checking)
  • Content cards benefit from thumbnails
  • Fewer taps = better for some user segments
