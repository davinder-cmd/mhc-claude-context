Web vs Mobile Navigation
Element Web Mobile
Primary nav Sidebar (left) Bottom tabs
Nav items 5 + More section 5 tabs
Collapse behavior 240px ↔ 72px N/A (always visible)
Settings access Sidebar "More" section Profile avatar menu
Toggle control ☰ in top bar N/A
Why sidebar on web, tabs on mobile:
  • Sidebar maximizes vertical content space on wide screens
  • Bottom tabs follow mobile platform conventions (iOS/Android)
  • Sidebar can accommodate more items (Settings, Devices, Help) without crowding
  • Collapsible sidebar gives user control over information density

5. Sidebar Specifications

Dimensions

Sidebar Expanded: - Width: 240px - Item padding: 12px 14px - Icon size: 24px (18px emoji) - Icon-label gap: 12px - Item font: 14px, 500 weight - Item radius: 8px - Section divider: 1px, 12px margin Sidebar Collapsed: - Width: 72px - Item padding: 12px (centered) - Icon size: 24px - Labels: Hidden (tooltip on hover) Transition: - Duration: 200ms ease - Properties: width, opacity (labels)

Top Bar (v3)

Top Bar: - Height: 64px - Background: white - Left: Toggle (40px) + Logo - Right: Avatar (36px) - No nav links (moved to sidebar) Toggle Button: - Size: 40px × 40px - Icon: ☰ (hamburger) - Hover: Warm background

6. Version Comparison

Feature v1 v2 v3
Web navigation Top nav (horizontal) Top nav (horizontal) Sidebar (collapsible)
Content max-width Fluid (fills viewport) 1200px centered 1200px centered
Chrome width Full viewport Full viewport Full viewport
Mobile nav Bottom tabs Bottom tabs Bottom tabs
Mobile content Full parity Full parity Full parity
Settings location (web) More menu (•••) More menu (•••) Sidebar "More" section
User preference — — Sidebar expanded/collapsed
When to use v3 (sidebar):
  • Users spend extended time in app (desktop power users)
  • Navigation needs to accommodate more items in future
  • Want to match patterns from productivity apps (Notion, Slack, Linear)
  • Users prefer persistent nav visibility vs. header space
When to use v2 (top nav):
  • Simpler mental model (same horizontal nav as mobile tabs)
  • Maximum horizontal content space
  • Less visual complexity
  • Closer to Personify's pattern (competitive familiarity)
